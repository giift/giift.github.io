Giift-SDK

For users

Installation

Add the following line to your Podfile:

pod 'Giift-SDK', :git => git@bitbucket.org:giiftalldev/giiftsdk-ios-uat.git', :branch => '1.x'

Usage

To run the project, clone the repo, run pod install from the ROOT directory first and open the Giift-SDK.xcworkspace.

Swift
  1. Create a bridging header: <app_name>-Bridging-Header.h
  2. Link in build settings
Target > Build Settings > Swift Compiler - General > Objective-C Bridging Header > dir/app_name-Bridging-Header.h

Requirements

iOS 9.0 and above

For developers

Environment

There are two methods to declare your environment. This should be done in the AppDelegate.

    GIIManager.sharedInstance.Environment = PRODUCTION;
    // OR
    GIIManager.sharedInstance.baseURL = "your-base-NSURL";
    GIIManager.sharedInstance.assetURL = "your-asset-NSURL";

    //initialize giift api credentials
    WebServiceController.sharedInstance.clientId = GIIFT_CLIENT_ID;
    WebServiceController.sharedInstance.clientSecret = GIIFT_CLIENT_SECRET;
    WebServiceController.sharedInstance.scopes = @[@"user",@"wallet",@"cart"];

Build

Select Giift-SDKFramework target and build it.

Test

Select GiiftTestingTests and test it.

Document

Install Appledoc.

Run appledoc --project-name Giift-SDK --project-company Giift.com Public
