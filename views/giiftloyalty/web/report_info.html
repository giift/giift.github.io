Getting information on a report

To generate reports, the user needs to know the report id and any query parameters that the report may require.

When the report id is not known, the user can get the list of all available reports, and then find the desired report.

If the report id is known, the user can get information about the specific report.

List

The /report/list method will return a list of all the reports the user has access to, and information coressponding to each report.

This includes the id and name of the report, and the query parameters needed to execute the report.

This response can be used to look for the desired report along with the necessary query parameters to generate that report, as described in the example below.

Info

Performing a GET request on the /report/info/<report_id> method returns the same information as the /report/list method, but only for a specific report.

An example of the response is shown below.

This method requires one uri parameter:

Parameter Description
report_id Id of the required report.

Example

To get the Age Distribution report, it is necessary to know if any query parameters are required.

If the id for this report is not known, the user will have to get a list of all the reports and find the id and query parameters corresponding to the Age Distribution report.

Executing the /report/info/agepie method returns the id, name and fields of this report.

As it can be seen, the parameters, 'from' and 'to' are mandatory, while the 'timezone' is not. The default values of the query parameters are also given.

There is another field, 'no_export', which is hidden and not required. The user does not need to provide any information regarding this parameter.

Using this information, the user can now continue to generate the report for Age Distribution.
