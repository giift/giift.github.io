Redeem a loyalty program

The idea behind the following requests is to debit points from a user loyalty program.

For each loyalty program, Giift.com stores the dollar value behond each point.

This dollar value or ratio points to cash is saved in Giift.com database in agreement with the merchant.

In simple words, it defines how much money the merchant is ready to give back to a loyal customer based on the customer's spending amount.

Perform a POST request to the following url: https://sandbox.giift.com/api/card/v1/debit.

Content-type sould be application/x-www-form-urlencoded.

List of parameters:
Parameter Description
card_id The loyalty card id. This is not the card number.
amt Amount in the program unit to be debited from the card.
type One of cash, points(default is points). If cash is chosen, it means you wish to calculate the dollar value behind the points being redeemed. For one program, there could be several ratios (2% cash back, 3% during holidays season). In Giift.com we refer to it as exchange. One exchange will bear the explicit ratios between two objects(ex:points program to cash). In this case, you must provide an exchange id. All exchange id can be retrieve using the method exchange/around

If points is chosen, it means your are just removing points from the card and no exchange_id is necessary.
exchange_id To be provided in case of type 'cash'.
